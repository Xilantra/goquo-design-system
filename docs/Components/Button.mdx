# Button
```import { Button, ButtonIcon, ButtonSkeleton } from '../src/components/Button/'```

## Introduction
Buttons express what action will occur when the user clicks or touches it. Buttons are used to initialize an action, either in the background or foreground of an experience. Use the ```<Button />``` or ```<ButtonIcon />``` components to render a primitive button, and ```<ButtonSkeleton />``` for the skeleton.<br/>
Do not use buttons as navigational elements. Instead, use ```Link``` when the desired action is to take the user to a new page. Default props for button is ```kind="primary"``` & ```scale="normal"```.


## Primary Button
Use it for the principle call to action on the page. Buttons are used primarily on action items. Some examples include Add, Save, Delete, and Sign up. Each page can have only **one or two primary buttons**. Any remaining calls-to-action should be represented as secondary buttons.

<Playground>
    <Button>Default</Button>
    <Button kind="primary" scale="normal">Click me</Button>
</Playground>


## Secondary Button
Use for secondary actions on each page.

<Playground>
    <Button kind="secondary" scale="normal">Click me</Button>
</Playground>


## Ghost Button
Use when an action does not require primary dominance on the page. Ghost buttons require a glyph icon (cannot be stand alone text because of poor affordance).

<Playground>
    <Button kind="ghost" scale="normal">Click me <svg width="16" height="16" viewBox="0 0 16 16" xmlns="http://www.w3.org/2000/svg"><path d="M7 7H4v2h3v3h2V9h3V7H9V4H7v3zm1 9A8 8 0 1 1 8 0a8 8 0 0 1 0 16z" fillRule="evenodd" /></svg></Button>
</Playground>


## Danger Button
Use when an action has harmful intentions to the users data (delete, remove, etc).

<Playground>
    <Button kind="danger" scale="normal">Click me</Button>
</Playground>

## Disabled Button
Use when the user cannot proceed until an input is collected. I’m not sure whether it’s ok to grey out the button instead of just fading the opacity. Keep in mind that disabled buttons exclude users with disabilities. It’s one of the best reasons to stop using them. The best practice is to keep the button enabled. Let the user click the button and then show the message why it can’t proceed.<br />
More on this: https://axesslab.com/disabled-buttons-suck/

<Playground>
    <Button kind="primary" scale="normal" disabled>Click me</Button>
</Playground>


## Icon Button
When words are not enough, icons can be used in buttons to better communicate what the button does. Icons are always paired with text for UX. The icon size must always be ```16px/1rem``` and the whole size need to have at least ```48px x 48px (3rem)``` in mobile.

## Link
Link with button style. Links are used primarily as a navigational element. Links may also change what or how data is displayed (view more, show all, etc). If the action taken by the user will change or manipulate data, use a button.

<Playground>
    <Button kind="primary" scale="small" as="a" href="https://goquo.com" target="_blank" rel="noreferrer noopener">Click me</Button>
</Playground>
